# 🚀 [5주차] 주간 회고 - KPT

## 🔄 KPT 회고

### 🍎 Keep (좋았던 점, 유지할 점)
> 잘 해냈거나, 팀 차원에서 계속 유지했으면 하는 점
- 분업 방식이 좋았다.
- 자유도 있는 개발 방식
- 아침에 점심 정하기
- 열정적으로 개발하기
- 데일리 스크럼이 신속했던 점
- 간식을 나눠먹는 따듯한 시간 * 3
- 꼼꼼한 설계
- 모든 면에서 딥다이브했던 점
- 대화가 많은 점
- 팀 분위기
- 서로 챙겨주는 모습
- 다들 코드스쿼드를 잘 적응한 점
- 개발을 드디어 시작한 점
- 팀원들의 배려 덕분에 회의가 잘 되었음
- 스트레스 없이 개발한 점
- 주변 사람들과의 좋은 네트워킹

### 🍏 Problem & Try (아쉬웠던 점, 문제점  +  시도할 점, 해결 방안)
> 불편했거나, 개선이 필요한 문제 상황
> Problem을 해결하기 위해 다음 주에 구체적으로 시도할 액션
- 쌓여있는 이슈 Edit으로 재활용하기
- 이야기가 분산되지 않도록 메인 주제는 다 같이 이야기 하기 (스택에 이야기할 것 쌓아두고 다 같이 시간될 때 언급하기)
- 월요일 아침 데일리 스크럼 끝나고 10분간 백로그 이야기 하기 + 금요일 회고 시간에 회고하기
- 작업 할당자(담당자)끼리 이야기하여 생산성 저하를 막는다. (API를 확인하여 담당자 확인)
- 내가 이슈를 제기하기 전에 상대방의 입장을 생각하기
- 이슈 제기할 때 슬랙을 통해 대화
    - 현실에서 노트북 접고 “저기…” 금지
- 회의때 다들 존댓말 쓰기
- 회의 전에 자신의 생각 모두 정리해서 텍스트로 남긴 후에 회의 시작하기
- 글과 말보다 그림으로 볼 수 있는 요소가 있으면, 서로의 생각을 이해하는데에 도움이 될 것 같다.
