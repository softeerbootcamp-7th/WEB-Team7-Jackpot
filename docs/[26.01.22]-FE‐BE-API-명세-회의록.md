## 📢 회의 안건
1. 로그인/회원가입 API 명세
2. 메인화면 API 명세 (통계, 캘린더, 최근 자기소개서, 알림)
3. 자료 업로드 API 명세 및 AI 라벨링 처리
4. 라이브러리 API 명세
5. 자기소개서 작성/수정 API 명세
6. 캘린더 API 명세
7. 첨삭 및 자료 업로드 추가 논의 필요 사항

## 🗣️ 논의 내용

### 1. 로그인/회원가입
- 로그인: JWT 방식 사용
- 회원가입: 중복확인 API, 회원가입 API 분리
- 사용자 이름 수정 불가

### 2. 메인화면 API

**통계:**
- 숫자만 반환

**캘린더:**
- 오늘이 포함된 3주 내역 제공
- 다가오는 일정 최근 3개 제공
- D-day 계산은 FE에서 처리

**최근 작성한 자기소개서:**
- 기업명/직무명: 볼드 처리
- 질문 유형, 라벨 리스트: 배지 형태

**알림:**
- 구성 요소: 이미지 타입, 보라색 문구, 볼드(기업명/직무명), 수정 전 문구
- 보라색 문구 중 볼드 위치 확인 필요
- 페이지네이션: FE에서 페이지 사이즈/페이지 넘버 전송
- SSE 방식: 새로운 첨삭/읽기/7일 후 삭제/새로운 분석/첨삭 삭제 → 안 읽은 알람 실시간 수신 (최근 7일 기준)

### 3. 자료 업로드

**API 구조:**
- 텍스트 API와 파일 API 분리

**AI 라벨링 화면:**
- '업로드한 자료를 분석하는 중입니다' 문구 제거
- AI 실패 가능성 존재 (약 10% 추정) - 에러 처리 필요
- 유저 플로우 및 UI 재활용 방안 추가 상의 필요

**에러 처리:**
- 응답 오류 시 에러 처리 방안 수립

### 4. 라이브러리

**기업 라이브러리:**
- 폴더 구조
- 왼쪽 리스트, 오른쪽 컨텐츠 영역

**문항 스크랩:**
- 문항 스크랩 개수 API
- 문항 스크랩 API - 탭 진입 시 호출
- 페이지네이션 적용 (내용 전체)

**질문 답변별 글자 수:**
- 질문 스트링 사이즈로 서버가 자동 계산하여 제공

**검색:**
- 검색 단위는 FE에서 결정 후 공유
- 오늘 구현 제외

### 5. 자기소개서 작성/수정

**자기소개서 작성(리스트):**
- 기업명/직무명: 볼드 처리
- 질문 유형, 라벨 리스트: 배지 형태

**자기소개서 등록:**
- 답변 입력: 빈 스트링으로 초기화
- 마지막 입력 시간 기록

**자기소개서 수정:**
- 자기소개서 ID, 만료 날짜를 첨삭 링크의 endpoint로 전달

**기타:**
- 스크랩 API
- 첨삭 링크 생성 endpoint

### 6. 캘린더

**날짜 클릭 시:**
- 시작 날짜부터 끝 날짜를 parameter로 전송하면 리스트 반환
- 초기 진입 시: 당일 날짜에 해당하는 공고만 표시 (최근 7개 X)

**공고 관리:**
- 공고 수정 API
- 공고 삭제 API

### 7. 추후 논의 필요 사항
- 첨삭하기 상세
- 자료 업로드 상세 플로우
- 피그마 디자인 수정
