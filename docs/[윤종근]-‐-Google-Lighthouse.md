# 📚 [학습 주제] Google Lighthouse

## 🎯 학습 목표
- 이 내용을 왜 학습했는가? 웹 페이지의 성능, 접근성, 검색 엔진 최적화(SEO) 상태를 객관적인 수치로 확인하고 개선해야 할 구체적인 지점을 찾기 위해 학습했다.
- 학습 후 기대 결과: Lighthouse 리포트를 해석하여 핵심 지표를 이해하고 점수가 낮은 원인을 파악하여 실제 웹사이트의 로딩  속도와 사용자 경험을 개선할 수 있다.

## ✍️ 상세 정리
### 개념 1: 4가지 측정 카테고리
설명: Chrome 개발자 모드의 Lighthouse 탭에서 검사를 하면 4가지 항목에 대해 0 ~ 100점의 점수로 환산하여 볼 수 있다.

- Performance: 웹 페이지가 얼마나 빨리 로딩되고 사용자와 상호작용 가능한지 측정한다.
- Accessibility: 시각 장애인 등을 위한 스크린 리더 호환성, 색상 대비, `alt` 태그 유무 등을 검사한다.
- Best Practices: HTTPS 사용, 보안 취약점, 최신 웹 표준 준수 여부를 확인한다.
- SEO: `meta` 태그, 적절한 폰트 크기, 크롤링 가능 여부 등 검색 노출에 영향을 주는 요소를 체크한다.

### 개념 2: 핵심 지표
설명: Lighthouse 성능 점수는 여러 지표의 가중 평균으로 이 지표들은 사용자가 느리다거나 불편하다고 느끼는 순간을 수치화 한 것이다.

- FCP(First Contentful Paint): 로딩 성능으로 브라우저가 첫 번째 텍스트나 이미지를 화면에 그리는 순간까지의 시간이다.
- LCP(Largest Contentful Paint): 로딩 성능으로 화면에서 가장 큰 이미지나 텍스트 블록이 다 그려지는 데 걸리는 시간이다.
- CLS(Cumulative Layout Shift): 시각적 안정성으로 로딩 중에 광고나 이미지가 갑자기 뜨면서 화면이 밀리는 정도이다.
- TBT(Total Blocking Time): 반응성으로 로딩 중 메인 스레드가 멈춰서 클릭이나 입력이 안 먹히는 차단 시간의 총합이다.
