## ✍️ 상세 정리
### 개념 1: 측정 기준의 종류
설명: 보통 많이 알려진 기준은 구문 커버리지, 결정 커버리지와 조건 커버리지이다.

- 구문 커버리지(Statement): 실행 가능한 코드 라인이 최소 한 번이라도 실행되었는지를 본다. 가장 달성하기 쉽지만 논리적 오류를 놓칠 확률이 있다.
- 결정 커버리지: `if`, `switch` 등의 조건문에세 `True`인 경우와 `False`인 경우 등 모든 가능한 경로가 실행되었지를 본다. 구문 커버리지보다 더 엄격한 기준이다.
- 조건 커버리지: 전체 결과가 무엇이 되든 상관하지 않는다. 오직 조건문 안에 있는 개별 조건식이 `True`와 `False`를 모두 경험했는지만 본다.

```Javascript
// 결정 커버리지, 조건 커버리지 비교
// 함수: 두 조건 중 하나라도 참이면 통과
function check(A, B) {
  if (A || B) {  // <--- 이 전체 결과(Decision)를 본다.
    return "Pass";
  } else {
    return "Fail";
  }
}

/* [결정 커버리지 100% 달성하기] */
// 목표: if문이 True가 되는 케이스와 False가 되는 케이스 만들기

// 1. A=True, B=False -> 전체 결과 True ("Pass" 실행)
// 2. A=False, B=False -> 전체 결과 False ("Fail" 실행)

// 결론: 전체 결과가 T/F 모두 나왔으므로 결정 커버리지 100%이다.
// (하지만 B가 True인 경우는 테스트되지 않았다.)

// ---------------------------------------------------

/* [조건 커버리지 100% 달성하기] */
// 목표: A를 T/F로, B를 T/F로 만들기 (전체 결과는 신경 안 씀)

// Test Case 1: A=True,  B=False 
// -> A는 True 경험, B는 False 경험. (전체 결과: True)

// Test Case 2: A=False, B=True
// -> A는 False 경험, B는 True 경험. (전체 결과: True)

/* [분석]
- A: True(Case1), False(Case2) -> OK
- B: False(Case1), True(Case2) -> OK
- 결론: 조건 커버리지는 100%이다.

- ★ 문제점: 두 케이스 모두 전체 결과는 'True'이다. 
  즉, 'Fail'을 반환하는 else 경로는 한 번도 실행되지 않았다.
  (조건 커버리지 100%지만 결정 커버리지는 50%인 상황)
*/
```
