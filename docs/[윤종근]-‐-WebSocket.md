# 📚 [학습 주제] WebSocket

## 🎯 학습 목표
- 이 내용을 왜 학습했는가? 자기소개서 첨삭 페이지 동시 접속과 수정 기능을 구현하기 위해 적합한 기술인지 판단하기 위해 학습했다.
- 학습 후 기대 결과: HTTP와 WebSocket의 차이를 이해하고 왜 실시간 수정/첨삭 기능에 WebSocket이 필수적인지 설명할 수 있다.

## 📖 핵심 내용 요약
- 핵심 개념 1: 단방향(HTTP) vs 양방향(WebSocket) - 요청 없이도 서버가 데이터를 보낼 수 있는 구조
- 핵심 개념 2: 핸드쉐이크(Handshake) - 최초 연결 시 HTTP 프로토콜을 WebSocket으로 업그레이드 시키는 과정

## ✍️ 상세 정리
### 개념 1: HTTP의 한계와 WebSocket의 필요성
설명: 기존의 HTTP는 클라이언트가 요청을 보내야만 서버가 응답하는 **단방향** 통신이다. 따라서 자기소개서 페이지에서 수정을 하면 첨삭자에게도 실시간 반영이 되어야 하므로 Polling 방식 등은 실시간성이 떨어지고 서버 부하가 있기 때문에 한계가 있다.

 반면, 웹 소켓은 HTML5에 등장한 실시간 웹 애플리케이션을 위해 설계된 통신 프로토콜로 TCP를 기반으로 한다. 신뢰성 있는 데이터 전송을 보장하고 메시지 경계와 순서가 보장된 양방향 통신을 제공한다. HTTP와 다르게 클라이언트와 서버 간에 최초 연결이 이루어지면 이 연결을 통해 양방향 통신을 지속적으로 할 수 있다. 이때 데이터는 패킷 형태로 전달되며 전송은 연결 중단과 추가 HTTP 요청 없이 양방향으로 이루어진다.

### 개념 2: WebSocket 핸드쉐이크
설명: WebSocket도 처음에는 HTTP 요청으로 시작한다. 클라이언트가 서버에게 WebSocket으로 업그레이드 요청(헤더에 Upgrade)을 보내면 서버가 승인(101 Switching Protocols)하면 그때부터 프로토콜이 변경된다.
