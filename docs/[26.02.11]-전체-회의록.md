### 문제 상황

첨삭 댓글 등록하는 경우에는 전문 x. 변경분(index?)으로 던진다.

첨삭 댓글이 달린 경우에는 

- 서버에서 반영 후 전문으로 양측에게 보낸다.
- 전문으로 온 먼저 확인한 후에 양측에 반영.
- → 이후에 입력 반영 진행.
- → 그러면 반영 전에 쓰던게 덮어써지는게 생길순 있는데? 반영 이전 변경 사항들을 모아놨다가 다시 처리하는 식의 방식으로 에러 안생기게 만드는 방법 찾아보기
- → 태그의 index 위치 계산이 틀어짐. 이상하게 들어갈 수 있다 (Redux, 동작의 순서를 보장해라)
- → 본문과 리뷰의 상태관리를 어떻게 할지와 순서 보장을 어떻게 할지에 대한 논의
